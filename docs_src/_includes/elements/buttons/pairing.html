Button Pairing

Use buttons in these combinations to communicate hierarchy of user action. The family (Core, Explore, Outlet, ect) color tone represents the intended primary call(s) to action.

Making groupings of buttons flow correctly

A singleton button possesses no margin

Buttons which have a previous sibling which is also a button receive a left margin of 8px, this accounts for the fact that buttons are display: inline-block which adds an additional 4px of space totaling 12px.

If your buttons will wrap prior to displaying block you will have an undesirable staircase. To rectify this place buttons in a container with the btn-group inline-group classes.

Primary with Secondary Button Theme

This pairing presents two actions of similar importance. Assign these roles to button pairings with the btn-primaryclass for primary buttons and btn-default for secondary buttons.

Primary with Secondary Button Link

Use when less emphasis is desired for the secondary CTA (Example: "Cancel"). Standalone link style (see Links Reference for more details) renders using all the same spacing rules as a normal button.
